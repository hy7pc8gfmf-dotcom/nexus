# Nexus Contributor License Agreement (CLA)

All contributors to Nexus must sign a Contributor License Agreement before their pull requests can be merged. This protects both the contributor and the project.

## 文件结构

| 文件 | 适用对象 |
|------|---------|
| `ICLA.md` | 个人贡献者（独立开发者、学生、自由职业者） |
| `CCLA.md` | 实体组织（公司、机构、团队）及其雇员 |

## 签署流程

### 个人贡献者 (ICLA)
1. 打开 `ICLA.md`
2. 填写签名栏（姓名、邮箱、GitHub ID、日期）
3. 发送到 `nexus-cla@cherryclaw.dev`，或在第一个 Pull Request 中附上扫描件/截图

### 实体组织 (CCLA)
1. 公司的授权代表打开 `CCLA.md`
2. 填写公司信息和签名
3. 发送到 `nexus-cla@cherryclaw.dev`
4. 每位实际贡献代码的雇员仍需签署 ICLA，但可在备注栏引用公司 CCLA 编号

## 为什么需要 CLA？

Nexus 采用 **Apache 2.0（公开版）+ 商业许可（企业版）** 双许可模式。CLA 确保：

- 贡献者的代码可以被 Apache 2.0 授权给全球用户
- 贡献者的代码也可以被纳入商业许可版本（授权给付费客户）
- 贡献者保留自己代码的版权（只授予使用权，不转让所有权）
- 项目免受专利索赔风险

## 协议的法律效力

CLA 是具备法律约束力的协议。如有疑问，建议咨询法律顾问后再签署。
